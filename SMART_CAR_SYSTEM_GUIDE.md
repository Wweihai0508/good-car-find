# 🚗 智能车行展示系统 - 使用指南

## 📋 系统简介

这是一个专为二手车行设计的智能展示系统，支持：
- 🏠 **本店车辆展示** - 展示你自己的库存
- 🤝 **周边车行资源** - 录入其他车行的车辆，带客服务
- 🎯 **智能推荐** - 根据客户需求快速匹配车辆
- 📱 **手机端快速录入** - 随时随地录入车辆信息

## 🚀 快速开始

### 1. 数据库升级

运行数据库升级脚本：
```bash
cd backend
mysql -u root -p your_database < database/enhanced_schema_v2.sql
```

### 2. 启动服务

```bash
# 启动后端
cd backend
npm run dev

# 启动前端（新终端）
cd frontend
npm run dev
```

### 3. 访问系统

- 前台展示：http://localhost:5173
- 管理后台：http://localhost:5173/admin
- 智能推荐：http://localhost:5173/smart-recommend

## 📱 功能使用

### 客户端功能

#### 1. 智能找车（推荐）
访问 http://localhost:5173/smart-recommend

**功能特点：**
- 快速筛选：价格范围、品牌、车型、燃油类型、年份
- 智能推荐：根据客户需求匹配车辆
- 双重车源：本店车辆 + 周边车行资源
- 需求记录：没有合适车辆时记录需求，有车通知

**使用场景：**
```
客户来到车行，询问："我想买一辆15万左右的SUV，最好是丰田的"
→ 打开智能推荐页面
→ 选择价格范围：10-20万
→ 选择品牌：丰田
→ 选择车型：SUV
→ 点击搜索
→ 系统展示本店的丰田SUV + 周边车行的丰田SUV
→ 带客户去看合适的车
```

#### 2. 车辆列表
访问 http://localhost:5173/cars

可以浏览所有车辆，支持基本的搜索和筛选。

#### 3. 车辆详情
点击任意车辆卡片查看详细信息，包括：
- 车辆基本信息
- 车辆图片
- 车况描述
- 联系方式

### 管理端功能

#### 1. 快速录入（手机端友好）
访问 http://localhost:5173/admin/quick-entry

**功能特点：**
- 移动端优化界面
- 快速录入其他车行的车辆信息
- 支持图片上传（最多9张）
- 自动记录车行信息

**使用场景：**
```
你走访周边车行，发现好车
→ 用手机打开快速录入页面
→ 拍照上传车辆图片
→ 填写车行名称、联系方式
→ 填写车辆基本信息
→ 点击提交
→ 车辆信息录入系统
→ 客户需要时，你可以带客户去看
```

**填写字段：**
- 车行名称 *（必填）
- 联系电话 *（必填）
- 联系人
- 详细地址
- 车辆品牌 *（必填）
- 车辆型号 *（必填）
- 年份 *（必填）
- 里程
- 价格（万元）*（必填）
- 车辆图片
- 备注

#### 2. 本店车辆管理
访问 http://localhost:5173/admin/cars

管理你自己车行的车辆，包括：
- 添加新车辆
- 编辑车辆信息
- 删除车辆
- 设置推荐/热门

#### 3. 数据统计
访问 http://localhost:5173/admin/statistics

查看车行统计数据：
- 车辆总数
- 本店车辆数
- 其他车行车辆数
- 销售统计

## 🎯 业务场景示例

### 场景1：客户上门咨询

**步骤：**
1. 客户来到车行，提出需求："想买一辆20万左右的轿车"
2. 你打开智能推荐页面
3. 设置筛选条件：价格10-20万，车型：轿车
4. 系统展示本店和周边车行的符合条件的车辆
5. 你带客户看本店的车，如果客户不满意，再带去周边车行
6. 如果成交，你获得佣金

### 场景2：走访周边车行

**步骤：**
1. 你去周边的"诚信车行"看车
2. 发现一辆2019年宝马3系，车况不错，价格28万
3. 用手机打开快速录入页面
4. 拍照上传车辆图片
5. 填写：车行"诚信车行"，电话139xxxx，联系人李总
6. 填写车辆信息：宝马3系，2019年，28万
7. 提交录入
8. 回到车行后，这辆车就显示在系统中了
9. 有客户需要时，你可以带客户去看

### 场景3：客户需求记录

**步骤：**
1. 客户想要一辆特定车型，但现在没有
2. 在智能推荐页面记录客户需求
3. 填写客户联系方式、预算范围、品牌偏好等
4. 后续录入新车辆时，系统可以自动匹配
5. 有合适车辆时通知客户

## 📊 数据结构说明

### 新增字段说明

**cars 表新增字段：**
- `source_type`: 车辆来源（self-自营车，other-其他车行）
- `dealership_name`: 车行名称
- `dealership_address`: 车行地址
- `dealership_phone`: 车行联系电话
- `contact_person`: 联系人
- `distance_from_shop`: 距离本店的距离（米）
- `location_description`: 位置描述
- `commission_rate`: 佣金比例（%）

**新增表：**
- `customer_browsing`: 客户浏览记录（用于智能推荐）
- `customer_requirements`: 客户需求记录
- `quick_entries`: 快速录入临时表

## 🐛 常见问题

**Q: 如何区分本店车辆和周边车行的车辆？**
A: 系统会自动标识，本店车辆显示"本店"，周边车行车辆显示车行名称和距离。

**Q: 佣金如何设置？**
A: 在录入车辆时可以设置佣金比例，成交后可以查看和统计。

**Q: 客户信息会记录吗？**
A: 可选。客户输入手机号后，系统会记录浏览历史和需求，用于精准推荐。

**Q: 可以删除录入的车辆吗？**
A: 可以。在管理后台可以删除或修改任何车辆信息。

**Q: 支持多少辆车行？**
A: 理论上无限制，建议先合作10-20家周边车行。
